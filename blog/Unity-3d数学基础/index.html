3、四元数

3.1 为什么使用四元数

欧拉角

概念：有三个角度(x,y,z)组成，在特定坐标系下用于描述物体的旋转量

欧拉角旋转约定

万向节死锁

欧拉角的优缺点

优点：

  • 直观，存储空间小
  • 可以从一个方向到另一个方向旋转大于180°的角度

缺点：

  1. 同一旋转的表示不唯一

    • 就是在Inspector面板中，不同的物体旋转(x+360°)所得的效果相同

  2. 万向节死锁

    • 当某个特定轴达到某个特殊值时，绕一个轴旋转 可能会覆盖住另一个轴的旋转，从而失去一维的自由度
    • Unity中X轴达到90度时，会产生万向节死锁

总结

因为四元数旋转不存在万向节死锁问题

所以往往用四元数来表示三维空间中的旋转信息

3.2 什么是四元数

四元数的构成

概念：四元数是由实数加上三个虚数单位组成，主要用于在三维空间中表示旋转

  • 四元数包含了一个标量和一个3D向量
    • [w,v] ，w为标量 ， v为3D向量
    • 可以展开为[w,(x,y,z)]
  • 对于一个四元数，就是表示3D空间中的一个旋转量

四元数基本公式

Quaternion：是Unity中表示四元数的结构体

轴角对方法初始化

四元数和欧拉角相互转换

四元数的优点

  1. 四元数的取值范围是180° ~ -180°，所以有同一位置，出现不同角度的情况
  2. 通过四元数去进行旋转，不会有万向节死锁的情况

3.3 四元数的主要方法

单位四元数

  • 单位四元数没有旋转量，当角度为0或者360时，对于给定轴就会得到单位四元数
  • [1,(0,0,0)] 和 [-1,(0,0,0)] 都是单位四元数。表示没有旋转量

  • 建议使用Slerp进行插值运算

总结

  1. 单位四元数：用于对象角度初始化
  2. 插值运算：用于平滑旋转
  3. 向量指向转四元数：用于让对象朝向某方向

3.4 四元数计算

四元数相乘

q3 = q1 * q2

  • 两个四元数相乘，得到一个新的四元数，代表两个旋转量的叠加，相当于旋转
  • 旋转的是本地坐标系
  • 使用四元数相乘改变物体旋转量

四元数乘向量

v2 = q1 v1(四元数在前，向量在后*顺序不能改变)

  • 向量乘一个四元数，相当于让该向量旋转 四元数的旋转量

4、MonoBehavior中的重要内容

4.1 延时函数

什么是延时函数

  • 就是会延时执行的函数，我们可以自己设定延时要执行的函数和具体延时的时间

  • 是MonoBehaviour基类中已经实现好了的方法

延迟函数的使用

延迟函数API：Invoke

注意事项
1、延时函数第一个参数填写的是字符串，第二个参数单位是秒

2、延时函数默认只能是无参的函数。需要经过一层函数的封装，才能调用有参的函数

3、延时函数默认只能调用自己对象的函数，同样是要经过一层函数的封装，才能调用其它对象

延迟重复执行函数API：InvokeRepeating

取消延迟函数API

  • 取消脚本上的所有延迟函数
  • 指定函数名取消

延迟函数受对象失活、销毁的影响

  • 脚本依附对象或脚本本身失活，并不会影响延迟函数
  • 脚本依附对象销毁或者脚本移除，延迟函数无法执行

总结

  • 将延迟函数 OnEnable( )或 Start( )中，对象激活时开启，比如计时等功能
  • 将取消延迟函数写在OnDisable( )中，对象失活时取消

示例：利用延时函数实现计时器

4.2 协同程序

Unity中的多线程

  • 使用多线程需要引入using System.Threading命名空间

  • Unity中关于对象的操作，只能在主线程中执行，新线程是无法访问的

  • Unity中开启的多线程，要手动关闭，否则会一直执行

Unity中多线程的作用

利用多线程来处理可能会造成主线程卡顿的问题

副线程中一般放入复杂的算法运算或者网络相关的内容，之后再将结果放到公共的内存中等待主线程的调用

协同程序是什么

协程是将代码分时执行，不卡主线程。就是把可能让主线程卡顿的耗时逻辑分时分布执行

主要使用场景

  • 异步加载文件
  • 异步下载文件
  • 场景异步加载
  • 批量创建时防止卡顿

协程和线程的区别

  • 每个线程都是一个独立的管道，和主线程并行执行
  • 协程是处于原线程之上开启的，进行逻辑分时分布执行

当执行协程函数时，会将代码分时分布的执行，未执行的代码会被挂起，直到协程执行完毕

协程的使用

继承MonoBehavior的类 都可以开启协程函数

  1. 申明协程函数
    • 函数返回值为 IEnumerator类型及其子类
    • 通过 yield return 后返回内容，决定多久后 将协程挂起
  1. 开启协程函数,一般在执行一次的生命周期函数中使用
    • 只能通过API调用：StartCoroutine( )
  1. 关闭协程
    • 关闭所有协程API：StopAllCoroutines()
    • 关闭指定协程API：StopCoroutine()

yield return 不同返回值的含义

协程受对象和组件失活销毁的影响

协程开启后

  • 组件和物体销毁，协程不执行
  • 物体失活协程不执行，组件失活协程执行

总结

  • Unity支持多线程，只是新线程无法访问主线程中Unity相关内容
    • 一般用于复杂的逻辑运算或网络消息的接收
    • 多线程一定要记得关闭
  • 协程不是线程，它是将线程中的逻辑进行分时执行，避免卡顿
  • 协程只有当组件单独失活时 不受影响，其它情况协程会停止

示例：计时器和许多的方块

示例一：使用协程实现计时器
示例二：创建十万个随机方块，不会明显的感觉到卡顿

5.4 Resources 资源卸载

Resources 重复加载资源不会浪费内存

  • 其实Resources 加载一次资源过后，该资源就一直存放在内存中作为缓存
  • 第二次加载时发现缓存中存在该资源，会直接使用
  • 所以 重复加载不会浪费内存，但每次加载需要查找和去除，会浪费性能

如何手动释放掉缓存中的资源

  • 卸载指定资源API：Resources.UnloadAsset 方法

    1. 该方法不能释放 已经实例化后的对象 以及GameObject对象
    2. 只能释放 不需要实例化的内容 比如 图片、音效等

  • 卸载未使用的资源API：Resources.UnloadUnusedAssets 方法

    • 一般在过场景时和GC一起使用

6、场景异步加载

场景同步切换

  1. 将创建好的场景拖入Build Settings 中
  2. 然后在脚本中使用API切换：SceneManager.LoadScene(场景名字符串)

场景同步切换的缺点

  • 在切换场景时，Unity会删除当前场景上的所有对象，并去加载下一个场景的相关内容
  • 如果当前场景对象过多，或下一场景对象过多，则整个过程会非常耗时

场景异步切换

场景异步加载和资源异步加载 几乎一致 有两种方式

  1. 通过事件回调函数 异步加载
  1. 通过协程函数 异步加载
    • 在切换场景时，会删除当前场景的所有对象和脚本，这就导致了协程函数不可用
    • 通过DontDestroyOnLoad 方法，可以让该对象过场景时，不被移除

  • Color：颜色变化
  • Corner Vertices：该属性值越大，角顶点就越圆滑
  • End Cap Vertices：该属性值越大，终端顶点越圆滑

  • Generate Lighting Data：默认不受光影响，当使用受到光照影响的材质球时，它不会显示
  • Materials：为线段设置材质球，默认无材质球

8、物理系统

8.1 范围检测

什么是范围检测

游戏中瞬时的攻击范围判断 一般会用范围检测

例如：玩家前方5m处释放一个地刺魔法，在此范围内的对象将受到伤害

以及，玩家攻击时，前方1米圆形范围内对象都受到伤害等等

类似这种没有实体的，只需要检测在指定某一范围是否让敌人受到伤害时，就使用范围判断

如何进行范围检测

想要被范围检测到的对象，必须具备碰撞器

范围检测相关API：当执行到该句代码时 进行一次瞬时的范围检测，并不会真正产生一个碰撞器，只是碰撞判断计算而已

简单来说，就是当我们调用该API时，它会瞬间检测该范围内是否有其它碰撞器

范围检测API

盒状范围检测：Physics.OverlapBox( )

  • 返回值：该范围内的所有触发器组件，可以用数组接收

  • 参数一：立方体中心点，盒状范围的原点

  • 参数二：立方体三边大小，从中心点开始，向两边分布构建立方体
  • 参数三：立方体角度，盒状范围旋转角度
  • 参数四：检测指定层级(不填则默认检测所有层)，LayerMask.NameToLayer(“层级名”)
  • 参数五：是否忽略触发器。默认UseGlobal-使用全局检测，Collide-检测触发器，Ignore-忽略触发器

另一个API：Physics.OverlapBoxNonAlloc( )

  • 返回值是 int类型数据：碰撞到的碰撞器数量
  • 参数：将创建好的碰撞器数组传入

球形范围检测：Physics.OverlapSphere()

  • 返回值：在该范围内的触发器

  • 参数一：中心点

  • 参数二：球半径
  • 参数三：检测指定层级，不填则检测所有层
  • 参数四：是否忽略触发器，默认UseGlobal-使用全局检测，Collide-检测触发器，Ignore-忽略触发器

另一个API：Physics.OverlapSphereNonAlloc()

  • 返回值：碰撞到的碰撞器数量
  • 参数：传入一个数组进行存储

胶囊范围检测：Physics.OverlapCapsule()

  • 参数一：半圆一 中心点
  • 参数二：半圆二 中心点
  • 参数三：半圆半径
  • 参数四：检测指定层级
  • 参数五：是否忽略触发器

另一个API：Physics.OverlapCapsuleNonAlloc()

示例：用WSAD控制立方体的前后移动和旋转，并进行范围检测

8.2 射线检测

什么是射线检测

在指定点 发射一个指定方向的射线，判断该射线与哪些碰撞器相交

  • 如FPS射击游戏(无弹道，不产生实际的子弹对象进行移动)

射线对象

  • 单独的一条射线并没实际意义，需要结合物理系统进行射线碰撞判断才有用

碰撞检测函数

射线检测也是瞬时的，仅在执行代码时进行一次射线检测

  • 射线是否碰撞到对象：Physics.Raycast()
    • 函数返回值为bool，如碰撞到对象，则返回true
    • 参数一：传入创建好的射线对象
    • 参数二：射线的最大检测距离，超过距离不检测
    • 参数三：检测指定层级
    • 参数四：是否忽略触发器
  • 获取相交的单个物体信息
    • 物体信息类 RaycastHit，该参数用来记录相交物体的信息，类型为结构体
    • 参数一：传入创建好的射线对象
    • 参数二：传入声明好的RaycastHit 对象
    • 参数三：射线的最大检测距离，超过距离不检测
    • 参数四：检测指定层级
    • 参数五：是否忽略触发器
  • 获取相交的多个物体：Physics.RaycastAll( ),返回值是一个数组，将碰撞到的多个物体倒序存入数组中
    • 参数一：传入创建好的射线对象
    • 参数二：射线的最大检测距离，超过距离不检测
    • 参数三：检测指定层级
    • 参数四：是否忽略触发器

注意事项

由于距离、层级两个参数都是 int类型

所以当我们将两者的顺序弄反时，编译器并不会报错

要记住，层级参数一定是在距离参数之后出现的
