图片导入设置

图片导入概述

Unity中使用最多的图片格式是：JPG、PNG、TGA 三种

图片设置的6大部分

  1. 纹理类型
  2. 纹理形状
  3. 高级设置
  4. 平铺拉伸
  5. 平台窗口
  6. 预览窗口

纹理类型设置

设置纹理类型主要是为了让纹理图片有不同的主要用途

  • default：默认贴图纹理，大部分导入的模型贴图都是该类型

    • sRGB：默认勾选，作用是让我们看到的颜色更自然
    • Alpha Source：指定如何生成纹理的Alpha通道
      • None：无论输入纹理是否有Alpha通道，导入的纹理都没有Alpha通道
      • Input Texture Alpha：输入纹理中的Alpha。适用于带Alpha通道的图片
    • Alpha Is Transparency：启用可以避免边缘上的过滤瑕疵

  • Normal map：法线贴图纹理

  • Editor GUI：编辑GUI

  • Sprite：2D游戏和UI相关内容

Sprite

SpriteEditor

SpriteEditor 是精灵图片编辑器，主要用于编辑2D游戏开发时 使用的Sprite精灵图片

作用：

主要用于编辑图集中提取元素、设置精灵边框、设置九宫格、设置轴心等功能

要将图片的类型设置为Sprite 才能使用

注意：

如果当前工程为3D工程的话，需要按照2D Sprite包才能使用SpriteEditor

SpriteEditor参数

Muleiple 图集元素分割

当图片资源为图集时，需要将模式设置为Multiple，就可以使用Sprite Editor对图集元素进行分割

详细参数移步文档

  • 进行图集元素切割时，一般使用Automatic(自动切割),再配合Trim手动调整，普遍性更高
  • Grid By Cell Size(按单元格大小分割),该模式适用于符合单元格规范的图集

Sprite Renderer

Sprite Renderer是精灵渲染器组件，2D游戏中的所有资源都是通过Sprite Renderer让我们看到(UI除外)

2D对象的创建

  • 直接拖入Sprite图片
    1. 可以将类型为Sprite的图片直接拖入场景中，就能完成创建
    2. 该图片的本质 其实就是空物体挂载了Sprite Renderer组件
    3. 而Sprite Renderer组件上的Sprite参数关联了拖入场景的图片，所以就能看到
  • 通过Hierarchy窗口 右键创建2D Object 中的Sprite
    1. 此时就会出现一个挂载了Sprite Renderer组件的空对象
    2. 将需要创建的图片，直接关联到该组件的Sprite中，即可完成创建

2D物理系统

2D刚体

参数面板相关：

Metarial：物理材质，一般不在刚体上设置，它会默认使用碰撞器的物理材质

Simulated：默认启动，能够模拟物理效果

Use Auto Mass：是否使用自动质量，默认不启动

Linear Dray：影响位移的阻力

Angular Dray：影响旋转移动的阻力

Gravity Scale：受重力影响的程度

Sleeping Mode：对象处于静止状态时进入睡眠模式

​ 1、Never Sleep：从不休眠，会一直进行检测

​ 2、Start Awake：最初处于唤醒状态，静止时休眠，适用于 移动物体

​ 3、Start Asleep：最初处于睡眠状态、被碰撞时唤醒。适用于 静止物体

如何选择不同类型的刚体

Dynamic 动态刚体：受力的作用，要动要碰的对象

Kinematic 运动学刚体：通过刚体API移动的对象，不受力的作用，会进行碰撞检测

Static 静态刚体：不动不受力作用的静态物体，会进行碰撞检测

刚体API

物理材质

物理材质是用于决定 在物体产生碰撞时的摩擦和弹性表现

通过物理材质，可以做出类型 斜坡不划落，小球反弹等效果

创建物理材质：

通过右键选择Create，找到Physics Material 2D，进行创建

物理材质的使用：

将创建好的物理材质，拖动到碰撞器的Material 参数中

恒定力

恒定力是一个特殊的脚本，它可以个2D刚体一个持续添加一个力

适用随着时间推移而加速的对象，比如火箭发射等效果

恒定力脚本会线性的为对象添加力和扭矩力

效应器

2D效应器一般配合2D碰撞器一起使用

可以让游戏对象在相互接触时产生一些特殊的物理作用力

比如传送带、互斥、吸引、漂浮、单向碰撞等效果

区域效应器：Area Effector2D

控制力的方向，可以实现传送带和向上弹簧的效果

浮力效应器：Buoyancy Effector2D

模拟流体行为，浮动和阻力相关设置，让该区域玩家看起来像在水里移动

点效应器：Point Effector 2D

模拟磁铁的排斥或吸引

将该效应器放到磁铁上，然后只对铁层级的物品有效，就能完成磁铁效果

平台效应器：Platform Effector 2D

2D游戏中的平台 或可往上跳跃的墙壁

表面效应器：Surface Effector 2D

表面效应器不需要勾选Is Trigger,主要用于模拟传送带

若要实现物体在传送到上打滚的功能，则勾选Use Contact Force

SpriteShape

SpriteShape是精灵形状的意思。

主要用于节约美术资源为前提，制作2D游戏场景地形 或者 背景

Tilemap

Tilemap一般被称为 瓦片地图或者平铺地图

主要用于快速编辑2D游戏中的场景

工作原理是 用小图排列组合成一张大地图

Tilemap的最小单位—瓦片

创建瓦片的方法

  1. Window-2D-Tile Palette瓦片调色板窗口创建
  2. 点击左边Create New Palette，然后选择存储路径
  3. 将需要的资源拖入到Palette窗口中，并选择需要保存的路径。就能成功创建瓦片

瓦片面板参数

瓦片调色板窗口

创建调色板窗口相关

  • Rectangle：矩形瓦片适合2D 横版游戏
  • Hexagon：六边形瓦片适合SLG策略游戏
  • Isometric：等距瓦片，可以进行上下移动
  • Isometric Z as Y：与前者类似，只不过多了一个Z轴，转化局部的Y轴，模拟伪Z轴

窗口操作技巧

编辑瓦片地图

  1. 创建好瓦片调色板窗口
  2. 将需要用到的所用地形素材全部拖入到调色板中
  3. 在Hierarchy面板中创建2D Object中的Tilemap，选择合适的瓦片网格布局
  4. 首先在调色板窗口点击Edit，启用编辑窗口
  5. 然后用吸管工具 吸取想要创建的地图，就可以直接在Scene窗口进行绘制

Isometric Z as Y 瓦片地图注意事项

  • 这种等距瓦片具有Z轴的概念
    1. 这就导致了瓦片图片和菱形地面具有一个高度差，两者不能贴合，这是图片的轴心点照成的
    2. 将图片的轴心点移动到 图片居中位置，就能解决高度差问题
  • 在3D游戏中，默认是由Z轴，来决定对象的深度，但2D游戏并没有Z轴，所以适用等距瓦片时要更改下默认的设置
    1. 在Unity编辑器的Edit窗口，选择Project Setting，再选择Graphics。最后将Transparency Sort Mode(排序模式)改为Custom Axis(自定义轴)进行排序
    2. 然后将下方Transparency Sort Axis 的参数改为(0,1,-0.26)
    3. 设置完参数后，将Tiemap Renderer脚本中的Mode(渲染模式)改为Individual
  • 也可以不通过更改轴心点的方式，解决高度差的问题
    1. 通过取消勾选下方的Lock Z Position
    2. 然后吸取图片后，就能通过加号和减号来控制高度

瓦片地图关键脚本和碰撞器

示例

使用等距瓦片创建地图，并实现人物在地图上的移动

注意事项：

人物导入之前

  • 需要将人物的轴心点改到图片人物的脚底位置

  • 然后将Sprite Renderer 中的Sprite sort Point(排序点)修改为Pivot，就能让人物底部出现在地面上

地图资源导入前

  • 需要将瓦片地图的轴心点移动到地图表面的中央，使其“下沉”,用于制作地面
    1. 将轴心点移动到瓦片地图下方的中央，可以使其“上抬”,用于制作墙体

等距瓦片的地图排序问题

  • 排序层相同时，会通过轴心点的Y轴进行渲染图片，轴心点低的渲染在前方，高的渲染在后方(排序方式一定要修改为Pivot)
  • 若排序层不同，则优选排序层的大小进行渲染，排序层大的渲染在前方，小的渲染在后方

等距瓦片地图碰撞相关问题

  • 若地图地面添加碰撞器，则人物则不能在表面移动。
  • 取消人物的重力，就能解决单个碰撞器下落的问题
  • 等距瓦片的碰撞类型选择Grid

Animation

打开Window->Animation-> Animation（或者Ctrl+6） 就能打开动画窗口

创建编辑Animation

创建动画

  • 在场景中选中想要创建动画的对象
  • 然后动画窗口就会出现一个Create
  • 点击创建后就能选择动画文件所保存的位置

当我们成功创建完一个动画后，Unity会做以下操作

1、对象上会多一个Animator组件

2、保存的位置中，会生成一个三角形的动画文件，以及方形的状态机文件

3、新生成的动画文件会被添加到状态机文件中

4、Animator组件中的Controller(状态机)参数 会自动关联对象

编辑动画

注意：编辑动画时，要注意一定要选中对象

  • 创建新动画

点击向下的箭头，可以创建新的动画。之后可以在这里进行动画切换

当我们新建动画时，Unity会做如下操作

1、在保存的位置创建一个动画文件

2、将新建的动画添加到之前的状态机中

  • 添加动画中需要变化的脚本(Add Property)

有限状态机

当满足某个条件时，就切换一个状态，且状态是有限的

动作系统和AI系统，都需要用到有限状态机

动画状态机

创建动画状态机

  1. 为场景中的物体创建动画时，会自动创建动画状态机(Animator Contorller)
  2. 也可以在Project窗口中右键创建Animator Contorller

动画状态机面板

Layers：动画层级页签，不同层级可以添加多个动画，权重(Weight)高的层级会覆盖低层级

可以通过点击 “+”号添加层级，不同层中可以有多个动画

不同层的动画可以同时播放

Parametes：参数页签，为动画状态机添加控制状态切换的参数

通过点击”+”号，可以添加 状态切换时的条件

  • Float：当大于，或者小于该数时，可以满足条件
  • int：当大于、小于、等于、或不等于该数时，可以满足条件
  • bool：判断T OR F ，满足条件
  • trigger：没有条件判断，点击后会自动切换回来，适用于来换切换状态的情况

添加动画切换条件

在Parameters参数页签中点击“＋”号，可以添加四种条件页签

1、在一个状态中点击右键，选择Make Transition，即可连接到另一个状态

2、然后点击左侧面板的”+”号，为状态添加条件页签

3、再点击Inspector面板中的”+” 号，添加切换条件

注意：一般要取消勾选Inspector面板中的Has Exit Time参数。这样就可以取消动画时的退出时间，达到流畅切换动画的效果

代码控制状态机的切换

Animator面板参数如下

  • Controller：对应的动画控制器

    默认在对象创建动画时，会自动关联

Animator中的API

2D动画

序列帧动画

原理是在一个循环中按一定时间间隔不停的切换显示图片

通过代码原理创建帧序列动画

思路：通过连续的更改sprite中关联的图片，达到帧序列动画的效果

方法：

1、创建一个2Dsprite

2、创建一个sprite数组，根据数组中填入的大小，用来关联所有图片

3、声明一个spriteRenderer 渲染图片，time记录时间，nowIndex记录数字索引

Animation窗口制作帧序列动画

方法1：

  1. 创建一个空物体

  2. 然后在Animation窗口中创建一个动画

    创建完毕之后，包含一个三角形的动画文件、方形的状态机文件、以及对象上关联的Animator脚本文件

  3. 选中三角形的动画文件，更改其动画帧率

    注意：可以通过修改动画帧率来控制动画的播放速度

    选中动画文件，在Inspector窗口中更改为Debug模式,其中就可以修改动画帧率

    注意：更改动画帧率之前，需要删除动画窗口中的关键帧

  4. 最后选中空物体，然后直接将某一个动作的帧序列拖入窗口中

方法2：

直接将某个一个动作的图片直接拖入到Hierarchy窗口中

该方法一般只用于制作第一个动画

注意：若出现无法将帧序列拖入到动画窗口中的情况，看看有没有选中物体

Animator切换动画

  1. 在Animator状态机中连接两个需要切换的状态

  1. 为两个状态添加条件页签

  1. 再选中切换连线，取消Has Exit Time参数，并添加切换条件

  1. 最后在脚本中，用代码控制动画切换
